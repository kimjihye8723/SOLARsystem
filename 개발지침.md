제공해주신 '태양광 모니터링 시스템 개발 지침서'의 구성 양식을 바탕으로, **Next.js(App Router) 환경에 최적화된 FMS(시설물 관리 시스템) 웹 프로그램 개발 지침서**를 작성했습니다. 퍼블리싱(UI/UX 프론트엔드 개발) 중심의 구현 원칙과 디자인 시스템을 반영했습니다.

# 시설물 관리 시스템(FMS) 개발 지침서

본 문서는 **시설물 관리 시스템(Facility Management System, FMS)** 프로젝트에 사용된 기술 스택, 시스템 아키텍처, 디렉토리 구조, 디자인 시스템 및 프론트엔드 퍼블리싱 구현 원칙을 정리하여 일관성 있는 개발과 유지보수를 돕기 위한 지침서입니다.

---

## 1. 프로젝트 개요 (Project Overview)

* **목적**: 건물 및 공장 내 자산 정보, 실시간 설비 상태(IoT 연동), 예방 정비(PM) 일정, 에너지 소비 현황을 통합 관리하고 관제할 수 있는 반응형 웹 애플리케이션입니다.
* **주요 특징**:
* 복잡한 대용량 설비 데이터를 직관적으로 구조화한 스플릿 뷰(Split View) 및 그리드 레이아웃.
* Tailwind CSS 기반의 테마 확장 및 다크 모드(Dark Mode) 지원.
* Next.js App Router 아키텍처 기반의 고성능 컴포넌트 구조 및 최적화된 클라이언트 사이드 렌더링(CSR) 활용.
* ApexCharts 및 Lucide React 아이콘 라이브러리를 활용한 모니터링 시각화.



---

## 2. 기술 스택 (Technology Stack)

### 2.1 Framework & Core

* **프레임워크**: `Next.js 14+ (App Router)`
* **기본 언어**: `TypeScript`, `JavaScript (ES6+)`
* **주요 역할**:
* App Router 기반의 직관적인 파일 시스템 라우팅 구현.
* 관제 시스템 특성에 맞춘 클라이언트 사이드 컴포넌트(`'use client'`) 중심의 실시간 UI 제어.
* Next.js 내장 최적화 기능(Image, Font)을 통한 정적 에셋 로딩 가속화.



### 2.2 Front-end UI & Publishing

* **스타일 시트**: `Tailwind CSS` (Utility-first CSS 프레임워크)
* **상태 관리 & 테마**: `next-themes` (다크 모드 전역 관리)
* **차트 및 데이터 시각화**: `ApexCharts` (or `Recharts` - Next.js 호환 클라이언트 컴포넌트)
* **아이콘 라이브러리**: `Lucide React` (Next.js 최적화 경량형 아이콘)
* **기본 웹폰트**: `Pretendard` (`next/font` 로컬 임포트)

## 4. 디자인 시스템 및 스타일 가이드 (Design System)

FMS 프로그램의 UI는 대용량 데이터를 다루므로 명확한 정보 위계와 즉각적인 상태 인지가 최우선입니다. 모든 스타일은 `tailwind.config.ts`에 정의된 시스템 컬러와 유틸리티 클래스를 사용합니다.

### 4.1 색상 및 상태 시스템 (Color Palette)

설비의 가동 상태를 직관적으로 구분하기 위해 명도와 채도가 대비되는 신호등(Traffic Light) 컬러 시스템을 적용합니다.

```typescript
// tailwind.config.ts 구조 예시
theme: {
  extend: {
    colors: {
      // 시스템 기본 테마
      brand: { primary: '#3b82f6', secondary: '#1e40af' },
      // 설비 상태 정의 컬러
      status: {
        normal: '#10b981',  // 정상 가동 (Green)
        standby: '#f5a623', // 대기 상태 (Orange/Yellow)
        stop: '#6b7280',    // 정지 상태 (Gray)
        fault: '#ef4444',   // 장애/고장 (Red)
      },
    },
  },
}

```

### 4.2 레이아웃 구성 원칙 (Grid & Split View)

* **대시보드 그리드**: 12컬럼 그리드 시스템(`grid grid-cols-12`)을 기본으로 활용하여 위젯의 중요도에 따라 크기를 유연하게 조정합니다.
* **설비 관리 스플릿 뷰**: 좌측에는 트리 구조 내비게이션(`w-64` 또는 `col-span-3`), 우측에는 상세 정보 테이블 및 카드(`flex-1` 또는 `col-span-9`)를 배치하는 스플릿 레이아웃을 일관되게 유지합니다.

### 4.3 다크 모드 (Dark Mode) 지원 원칙

* `next-themes` 플러그인을 활용하여 최상위 HTML 태그에 `.dark` 클래스를 토글합니다.
* 클래스 기반 스타일링을 사용하여 다크 모드 시 배경색과 텍스트 색상이 자연스럽게 반전되도록 처리합니다. (예: `bg-white dark:bg-slate-900 text-slate-900 dark:text-white`)

## 5. 프론트엔드 퍼블리싱 구현 세부 규칙

### 5.1 Next.js 파일 가이드

* **클라이언트 컴포넌트 명시**: 차트(`ApexCharts`), 캘린더, 실시간 토스트 알림, 드롭다운 등 사용자의 인터랙션이 들어가는 모든 컴포넌트 및 페이지 최상단에는 반드시 `'use client'` 지시어를 명시해야 합니다.
* **시맨틱 마크업**: Next.js 환경에서도 레이아웃 구조화 시 `<header>`, `<aside>`, `<main>`, `<section>`, `<article>` 태그를 의미에 맞게 사용합니다.

### 5.2 UI 컴포넌트 퍼블리싱 규칙

* **데이터 테이블(Data Table)**: 대용량 설비 리스트 조회를 위해 테이블의 헤더 고정(`sticky top-0`), 가로/세로 스크롤바 스타일링, 행 마우스 오버 피드백(`hover:bg-slate-50`)을 필수로 구현합니다.
* **도면 뷰 인터랙션**: `monitoring` 페이지의 공간 도면은 해상도에 깨지지 않도록 SVG 포맷을 활용하며, 각 설비 포인터에 펄스 애니메이션 효과(`animate-ping`)를 주어 장애 발생 시 시각적 임팩트를 부여합니다.

### 5.3 차트 및 에셋 연동 규칙

* **ApexCharts SSR 우회**: Next.js 빌드 시 브라우저 전용 객체(window) 참조 에러를 방지하기 위해, 차트 컴포넌트는 `next/dynamic`을 이용하여 SSR을 비활성화하고 동적 임포트 처리합니다.
```typescript
import dynamic from 'next/dynamic';
const Chart = dynamic(() => import('react-apexcharts'), { ssr: false });

```

## 6. 개발 및 실행 지침

### 6.1 로컬 실행 환경 구동

본 프로젝트는 **Next.js** 기반이므로 Node.js 환경이 구축되어 있어야 합니다.

1. **의존성 라이브러리 설치**:
```bash
npm install

```


2. **로컬 개발 서버 실행**:
```bash
npm run dev

```


3. **접속 주소**:
* 실행 후 터미널에 표시되는 로컬 주소(기본값 `http://localhost:3000`)에 접속합니다.
* 루트 경로 진입 시 설정된 라우팅 가이드에 따라 자동으로 종합 대시보드(`/dashboard`) 화면으로 연결됩니다.



### 6.2 신규 기능 및 페이지 추가 절차

1. **App 폴더 내 라우팅 생성**: 신규 관제 화면이 필요할 경우 `src/app/` 하위에 새로운 폴더(예: `energy/`)를 생성하고 `page.tsx` 파일을 작성합니다.
2. **공통 네비게이션 업데이트**: `src/components/layout/Sidebar.tsx` 파일 내의 메뉴 리스트 배열에 신규 페이지 경로와 Lucide 아이콘을 추가하여 전역 내비게이션을 동기화합니다.
3. **반응형 대형 디스플레이 검증**: 관제실 모니터(FHD, QHD 이상) 환경에서 대시보드 컴포넌트가 흐려지거나 깨지지 않는지 뷰포트 스케일을 확인합니다.
